# Product Field Map

This document maps existing Sortly/exported inventory fields to the C&C Feed Inventory app data model.

The goal is to preserve useful product/inventory data while cleaning it into fields the MVP can actually use.

---

## Current Source

Likely source:

- Sortly CSV export

Current known behavior:

- Inventory value appears to be calculated from price × new quantity.
- Barcodes/QR codes are not currently used in daily workflow.
- Product photos may be useful, but importing them may need separate handling.

---

## MVP Product Fields

| App Field | Purpose | Required for MVP | Notes |
|---|---|---:|---|
| product_id | Internal unique ID | Yes | Generated by app/database |
| product_name | Display name | Yes | Main product label |
| category | Product grouping | Yes | Salt, Mineral, Tubs, Blocks, Supplement, etc. |
| current_quantity | Current available quantity | Yes | Used for inventory counts |
| minimum_quantity | Low-stock threshold | Yes | Used for Low Stock report/status |
| unit_label | Unit name | Yes | Examples: units, bags, tubs, blocks |
| sale_price | Price charged/valued per unit | Yes | Used for invoice/use records |
| cost_per_unit | Internal cost | Optional | Role-restricted; Admin/Manager only |
| inventory_value | Calculated value | Yes | current_quantity × sale_price |
| vendor | Supplier/source | Optional | Useful later |
| source_notes | Notes from current system | Optional | Preserve if available |
| product_photo_url | Product image | Optional | Useful but not required for first logic build |
| status | Active/archived | Yes | Keeps old products from cluttering active lists |

## Suggested Sortly → App Mapping

| Sortly / Export Field | App Field | Notes |
|---|---|---|
| Name / Item Name | product_name | Clean for consistency |
| Folder / Category | category | Normalize categories |
| Qty / Quantity / New Qty | current_quantity | Confirm exact source column |
| Min Qty / Minimum Quantity | minimum_quantity | If missing, set manually |

## Import Cleanup Rules

### Product Names

Product names should be cleaned for consistency.

Avoid duplicates caused by:

- inconsistent capitalization
- trailing spaces
- abbreviations
- slightly different wording

Example cleanup:

- Garlic salt block
- Garlic Salt Blocks
- garlic salt blocks

Should become:

- Garlic Salt Blocks

---

### Categories

Categories should be controlled values.

Initial category options:

- Salt
- Mineral
- Tubs
- Blocks
- Supplement
- Other

These can be refined after reviewing real product data.

---

### Quantities

Current quantity should be numeric.

If quantity is missing:

- flag for review
- do not guess unless confirmed

If quantity is zero:

- keep as zero
- do not remove automatically

---

### Sale Price

Sale price is used for invoice/use record totals.

If sale price is missing:

- allow product to exist
- prevent invoicing until price is added
- show Missing Price warning

---

### Inventory Value

Inventory value should be calculated by the app:

current_quantity × sale_price

Do not rely on imported value as the source of truth.

---

### Product Photos

Product photos are useful for fast field use.

For MVP:

- keep photo field optional
- support placeholder image/card if missing
- import manually later if Sortly photo export is messy

---

### Barcode / QR

Barcode and QR fields are not MVP.

Reason:

Current users do not appear to rely on barcode/QR scanning, and adoption depends on making the workflow easier than ignoring it.

---

## Required Product Data Before Build

Before real database import, confirm:

- final product names
- current quantities
- sale prices
- low-stock minimums
- categories
- whether photos are available
- whether old/inactive products should be archived
