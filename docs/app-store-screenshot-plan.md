# StockLog App Store Screenshot Plan

This plan prepares screenshot capture for App Store Connect and TestFlight marketing/review materials.

## Required iPhone Screenshot Sizes

Prepare screenshots for the device sizes App Store Connect requires at submission time. Apple requirements can change, so confirm in App Store Connect before final capture.

Common iPhone screenshot sets to prepare:

- 6.9-inch iPhone display screenshots for recent Pro Max devices.
- 6.7-inch iPhone display screenshots if requested.
- 6.5-inch iPhone display screenshots for older large devices if requested.
- 5.5-inch iPhone display screenshots if App Store Connect requires legacy support.

Practical approach:

1. Capture the latest required large iPhone size first.
2. Add smaller required sizes only if App Store Connect asks for them.
3. Keep the same screen sequence and captions across sizes.

## Recommended Screenshot Sequence

Top sequence:

1. Login / welcome
2. Dashboard
3. Inventory list
4. Take Feed
5. Invoice detail
6. Reports
7. Manage Users / invites

## Demo Data Rules

- Use clean demo data only.
- Do not show real customer names, phone numbers, emails, addresses, invoices, payments, or private notes.
- Do not show private product/customer CSV export data.
- Avoid test records with jokes, placeholders, or rough internal names.
- Use plausible but fictional ranch/customer/product names.
- Keep quantities and prices realistic but not sensitive.

## Visual Direction

- Use warm StockLog styling.
- Show clear readable screens with high-value workflows.
- Keep screenshots uncluttered.
- Prefer screens with real operational value over empty states.
- Avoid private customer information.
- Avoid showing service-role keys, invite hashes, debug logs, console output, or local file paths.
- Confirm mobile safe-area headers, back buttons, profile icon, and bottom nav are visually clean.

## Capture Notes

- If a screen contains pending invite codes, use demo-only codes and avoid showing any live onboarding code.
- For invoice/payment screens, use fictional customer data and small demo amounts.
- For Manage Users, show role variety without real emails.
- For Reports, use demo data with meaningful totals.
